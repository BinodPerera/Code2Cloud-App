## Requirements

Before you begin, ensure you have the following installed on your system:
- **Python 3.8+** (recommended)
- **pip** (Python package installer)
- **MongoDB** database (either local run or MongoDB Atlas)
- **GitHub OAuth App** credentials (for authentication)



## Setup Instructions

### 1. Navigate to the backend directory
Open your terminal and make sure you are in the `backend` directory of the project:
```bash
cd /path/to/Code2Cloud/backend
```

### 2. Create a Virtual Environment
It's highly recommended to use a virtual environment to isolate the project dependencies.
```bash
python3 -m venv venv
```

### 3. Activate the Virtual Environment
Activate the environment before installing dependencies:
- **On macOS/Linux:**
  ```bash
  source venv/bin/activate
  ```
- **On Windows:**
  ```bash
  .\venv\Scripts\activate
  ```

### 4. Install Dependencies
Install all required packages from `requirements.txt`:
```bash
pip install -r requirements.txt
```

*(Alternatively, you can run `./venv/bin/pip install -r requirements.txt` directly without activating the virtual environment).*

### 5. Set Up Environment Variables
The application requires several environment variables to function properly, including database connection strings and OAuth secrets.

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```
2. Open the `.env` file and fill in your actual values:
   - `GITHUB_CLIENT_ID`: Your GitHub OAuth App Client ID
   - `GITHUB_CLIENT_SECRET`: Your GitHub OAuth App Client Secret
   - `GITHUB_REDIRECT_URI`: Usually `http://localhost:8000/api/v1/auth/github/callback` for local development
   - `SECRET_KEY`: A strong, random string (e.g., generated via `openssl rand -hex 32` or similar)
   - `MONGODB_URL`: Your MongoDB connection string
   - `DATABASE_NAME`: The name of your database (e.g., `code2cloud`)

## Running the Application

### Starting the Development Server
You can start the FastAPI server using `uvicorn` with the `--reload` flag for auto-reloading during development:

```bash
uvicorn main:app --reload
```
*(Or use `./venv/bin/uvicorn main:app --reload` if you haven't activated the virtual environment).*

### Accessing the API
Once the server is running, the application will be available locally. You can access:
- **The API Base URL:** `http://localhost:8000`
- **Interactive API Documentation (Swagger UI):** `http://localhost:8000/docs`
- **Alternative API Documentation (ReDoc):** `http://localhost:8000/redoc`
