# GCP Compute Engine Operations Guide (Frontend)

This guide provides instructions for managing the frontend application and Nginx Proxy Manager (NPM) deployed on a Google Compute Engine `e2-micro` instance.

---

## 1. Accessing the Services

After deployment, you can access the following:

- **Frontend Application**: `http://[STATIC_IP]` (Once configured in NPM)
- **NPM Management UI**: `http://[STATIC_IP]:81`
    - **Default Username**: `admin@example.com`
    - **Default Password**: `changeme`

> [!IMPORTANT]
> Immediately change the default NPM credentials after your first login.

---

## 2. Managing the VM via CLI

### SSH into the Instance
To access the VM terminal:
```bash
gcloud compute ssh code2cloud-frontend-vm --zone us-central1-a
```

### View Application Logs
Once SSH-ed into the VM, navigate to `/opt/npm` to manage the containers:
```bash
cd /opt/npm

# View logs for all services
docker compose logs -f

# View logs for the frontend specifically
docker compose logs -f frontend
```

### Restarting Services
```bash
# Restart everything
docker compose restart

# Restart only the frontend
docker compose restart frontend
```

---

## 3. Configuring NPM for Frontend

1. Log in to the NPM Management UI (`http://[STATIC_IP]:81`).
2. Go to **Hosts** > **Proxy Hosts**.
3. Click **Add Proxy Host**.
4. **Domain Names**: Enter your domain (or leave blank if just using IP for now, though NPM usually requires a domain).
5. **Forward Hostname / IP**: `frontend` (This is the service name in `docker-compose.yml`).
6. **Forward Port**: `80`.
7. Click **Save**.

---

## 4. Troubleshooting

- **Containers not running**: Check status with `docker ps`.
- **NPM not accessible**: Ensure port 81 is open in the GCP Firewall (Terraform should have handled this).
- **Static IP**: The external IP is static and won't change on reboot.
