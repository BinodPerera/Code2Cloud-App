# GCP Cloud Run Operations Guide

This guide provides instructions for managing the frontend application deployed on Google Cloud Run.

---

## 1. How to Login to GCP Project

To access GCP resources via the CLI, you need to authenticate and set your active project.

### Step 1: Login via CLI
Run the following command to authenticate with your Google account:
```bash
gcloud auth login
```
This will open a browser window for you to select your account.

### Step 2: Set Project ID
Set your active project to the one managing your Cloud Run services (e.g., `your-project-id`):
```bash
gcloud config set project [PROJECT_ID]
```

---

## 2. How to "SSH" into Cloud Run

> [!IMPORTANT]
> **Google Cloud Run is a Serverless platform.** Traditional SSH access into a running container is **not supported** because containers are stateless and scale to zero.

### Alternatives for Troubleshooting:

#### A. View Application Logs
To see stdout/stderr and error traces:
```bash
# View recent logs
gcloud logging read "resource.type=cloud_run_revision AND resource.labels.service_name=[SERVICE_NAME]" --limit=20
```
Alternatively, view logs in the **GCP Console** under Cloud Run > [Service Name] > Logs.

#### B. Local Container Testing (Recommended)
If you need to explore the file structure or environment of the container, run it locally:
```bash
# Pull and run your image locally for debugging
docker run -it [IMAGE_URI] /bin/sh
```

## 3. How to View Cloud Run .env Variables

Environment variables are configured within the Cloud Run service revision.

### Method 1: Via CLI
To describe the service and list all environment variables:
```bash
gcloud run services describe [SERVICE_NAME] --region=[REGION]
```
Replace `[SERVICE_NAME]` with your service name (e.g., `code2cloud-frontend`) and `[REGION]` with your region (e.g., `us-central1`).

### Method 2: Via GCP Console
1. Navigate to **Cloud Run** in GCP Console.
2. Click on your service (e.g., `code2cloud-frontend`).
3. Go to the **Variables** tab (or **Revisions** > **Container** to inspect configuration).
